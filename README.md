To consume tokens from a React component:

```scss

    padding: var(--gap-card);

```

Components **never** reference primitive tokens ÔÇö always semantic CSS variables (`var(--color-brand)`, not `$brand-500`). See `../docs/component-standards.md` ┬ž3.5.

## Forking this library for your project

**Do not** `npm install bleizlabs-ui` ÔÇö this library follows copy-to-project pattern (per D2).

```bash
# In your consumer project
cp -r path/to/bleizlabs-ui/dev/styles your-project/styles
cp -r path/to/bleizlabs-ui/dev/components your-project/components  # Phase 1+
```

Then edit `your-project/styles/_project-settings.scss` ÔÇö change seed values (brand color, accent, radius, spacing defaults). The generator cascades every change through the entire design system automatically.

## Architecture

- **SCSS Modules only** (D1) ÔÇö no Tailwind, no CSS-in-JS
- **Copy-to-project** (D2) ÔÇö no runtime dependency
- **Seed-based tokens** (D3) ÔÇö 5-10 seeds generate the full system
- **Fully styled** (D4) ÔÇö not headless, but fully themed via semantic tokens
- **Zero runtime UI dependencies** (D5 updated, D25) ÔÇö including complex interactive (Dialog, Popover, Tooltip, etc.), built from scratch against WAI-ARIA APG
- **Shadcn-aligned naming** (D24) ÔÇö flat pattern (CardHeader not Card.Header, Input not InputField)
- **Spacing scale** (D9) ÔÇö Tailwind-style 4px unit (index ├Ś 4 = px), indexes 0-20

Full rationale: `../docs/decisions.md`.
