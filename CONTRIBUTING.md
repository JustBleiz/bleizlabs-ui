# Contributing

Thank you for your interest in `@bleizlabs/ui`.

## Current status

This library is **open-source for visibility but not currently accepting external
contributions**. The codebase is published publicly so engineers can browse the
component patterns, accessibility implementations, and SCSS token architecture,
but pull requests will not be reviewed or merged at this time.

## What you can do

- **Browse the source.** Every component, every test, every token is in this repo.
- **Fork freely.** The MIT license permits any use, modification, or redistribution.
  If you fork, please be aware that you own your fork — upstream changes will not
  propagate automatically.
- **Open issues for bugs you spot.** Reports are welcome but please understand
  there is no SLA on response times. Issues that include a minimal reproduction
  are most useful.
- **Open issues for security findings.** See [SECURITY.md](SECURITY.md) for the
  preferred disclosure channel.

## What we cannot accept

- **Pull requests.** PRs from external contributors will be closed without review.
  This may change in the future, but for now the maintenance bandwidth is reserved
  for internal product use cases.
- **Feature requests.** The component roadmap is driven by internal product needs.
  Useful patterns from real consumer projects may be promoted upstream over time
  (per the Rule of Three), but external feature requests do not enter that
  prioritisation.

## If you want to use the library

See [README.md](README.md) for installation and usage. The library is distributed
via GitHub Packages, which requires a personal access token with `read:packages`
scope to install.
